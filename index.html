EasyTimepicker for jQuery Mobile

Overview

EasyTimepicker for jQuery Mobile allows developers to simply and quickly include timepicker controls in their jQuery Mobile projects. No javascript is needed to display the timepickers on the page, rather they are specified by means of <input type="time" /> html elements.

EasyTimepicker works by augmenting the input element with a select element that displays time entries generated with a specified time interval. So, for example, with a time interval of 15 minutes specified, the select will show time entries 00:00, 00:15, 00:30, 00:45, 01:00, etc. Any interval in the range 1 to 1440 may be specified, with 1440 being the number of minutes in a day. The original input element specified by the developer is not removed, but is simply hidden on the page; any selections made on the timepicker are immediately updated in the underlying input element and any change to the input element with the jQuery val() method are likewise updated to the select. If the input is updated with a time value that falls in between two time entries on the select, then this value is rounded to the nearest valid time entry and the rounded value is updated to both input and select elements. In this way the programmer can interact with the timepicker via the input element specified in the html source and EasyTimepicker transparently handles the display of the timepicker.

Examples

  • Example A

    This first example shows a timepicker defined with a simple input element and no parameters specified. Because no default time is specified in the input element's value attribute, the timepicker will default to the current time. The time entries will be displayed in 12-hour format, but the values in the select will be 24-hour format. The timepicker defaults to a 15 minute interval when generating the time entries in the list.

    Code

    <input type="time" id="exampleA" />

    Preview

  • Example B

    This example shows a timepicker defined with a default time value of 08:00. It uses the attribute data-native-menu="false" to display the select as a jQuery Mobile custom menu instead of a native select. The attributes data-display-24hour="true" and data-value-24hour="false" set the display to 24-hour format and the values returned to 12-hour format. This is the inverse of the default settings as shown in Example A. The time interval has been set to 30 minutes.

    Code

    <input type="time" id="exampleB" value="08:00" data-native-menu="false" data-display-24hour="true" data-value-24hour="false" data-interval="30" />

    Preview

  • Example C

    This example shows a timepicker defined with a default time value of 04:00 PM and having an interval of 120 minutes (2 hours). It explicitly sets the attribute data-value-24hour to "true" although, as mentioned earlier, this is the default setting, so even without this setting the values returned would be in 24-hour format. But the attribute data-display-24hour="true" is necessary for the timepicker entries to be displayed in 24-hour format.

    This example illustrates two points. Firstly, the timepicker can be updated with times in either 12-hour or 24-hour formats, irrespective of the format used for either the display or value returned. Secondly, it shows how the input element can be refrerenced by a label element and EasyTimepicker will then alter the for=".." attribute of the label to associate it with the corresponding timepicker.

    Code

    <div data-role="fieldcontain">

    <label for="exampleC">This is Example C</label>

    <input type="time" id="exampleC" value="04:00 PM" data-display-24hour="true" data-value-24hour="true" data-interval="120" />

</div>

Interaction

The developer can interact with the timepickers on the page using the <input type="time" /> elements defined in the page.

Query

So, for example, we could query the value of the timepicker in Example A with the following code:

<input type="button" value="Show the selected value for Example A" onclick="alert($('#exampleA').val())" />

Update

The jQuery method val() may be called on the input element to update the associated timepicker. The update value must be a valid time in either 24-hour or 12-hour format. Updates with invalid time values will be ignored and the update cancelled. So, for example, the following code will update Example A to show 02:30 PM.

$('#exampleB').val('14:30')

Try it yourself! Fill in a time and click on the corresponding button to update the timepickers in the examples.

Usage

Including javascript files

The code for EasyTimepicker for jQuery Mobile is divided into two separate files. The first, jquery.easytimepicker.js, implements the timepicker as a standard jQuery plugin. This can be used independantly from jQuery Mobiile and allows any <input> element to be substituted for a timepicker. The second file, jquery.easytimepicker.mobile.js, makes use of the jquery plugin and provides the functionality of replacing all <input type="time"> elements in the jQuery Mobile pages with timepickers.

So to use EasyTimepicker in a jQuery Mobile project just include these two javascript files after the includes for jQuery and jQuery Mobile, as follows:

<script type="text/javascript" src="jquery.easytimepicker.js"></script>

<script type="text/javascript" src="jquery.easytimepicker.mobile.js"></script>

Alternatively, the file jquery.easytimepicker.combined.js can be included, which combines the two sources into a single file. This javascript in this file has been minified with YUI Compressor 2.4.4. This demo makes use of the combined source file.

Once the required files have been included, <input type="time"/> is all that is needed to define a timepicker. An id is not even required and if it doesn't exist, then EasyTimepicker will generate a unique id and assign it to the input element. But, obviously, to simplify interaction with the timepicker, it is best to give it an id! ;)

Available attributes

The following attributes can be defined in the <input type="time"> element to control the behaviour of the timepicker:

  • data-native-menu Valid values: true / false
    Specifies whether the selects must be built using jQuery Mobile's data-native-menu="true" option
    Default value: true
  • data-interval Valid values: 1 to 1440
    Specifies the interval to use when generating the time entries
    Default value: 15
  • data-display-24hour Valid values: true / false
    A value of true indicates that the time entries must be displayed in 24-hour format, otherwise 12-hour format is used.
    Default value: false
  • data-value-24hour Valid values: true / false
    A value of false indicates that 12-hour format must be returned on val() of input elements with associated timepickers, otherwise 24-hour format is used.
    Default value: true
